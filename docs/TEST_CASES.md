# Rogue Quest 测试用例

## 1. 生物系统测试

### TC-001: 添加新生物
**前置条件：** 新游戏开始
**步骤：**
1. 完成第一关
2. 选择任意生物
**预期结果：**
- 生物添加成功
- 控制台显示 "添加成功，当前队伍: [creatureId]"
- 下一场战斗开始时显示英雄+1个生物

### TC-002: 添加同种族生物触发羁绊
**前置条件：** 已有1个城堡生物
**步骤：**
1. 完成第二关
2. 选择另一个城堡生物
**预期结果：**
- 生物添加成功
- 控制台显示 "羁绊激活: castle 等级X (数量:2)"
- 下一场战斗正常开始
- 英雄+2个生物都显示在战场上

### TC-003: 生物升星
**前置条件：** 已有1个生物
**步骤：**
1. 完成关卡
2. 选择相同生物（显示 ★1 → ★2）
**预期结果：**
- 升星成功
- 控制台显示 "升星成功: [name] → ★ 2"

### TC-004: 队伍满后只显示升星选项
**前置条件：** 已有5个生物
**步骤：**
1. 完成关卡
2. 查看肉鸽选项
**预期结果：**
- 只显示可升星的生物
- 不显示新生物

### TC-005: 满星生物不再出现
**前置条件：** 某个生物已3星
**步骤：**
1. 完成关卡
2. 查看肉鸽选项
**预期结果：**
- 该生物不会出现在选项中

## 2. 羁绊系统测试

### TC-101: 2个同种族触发羁绊
**输入：** 2个城堡生物
**预期：** 攻击+5%, 防御+5%

### TC-102: 3个同种族触发羁绊
**输入：** 3个城堡生物
**预期：** 攻击+10%, 防御+10%

### TC-103: 5个同种族触发羁绊
**输入：** 5个城堡生物
**预期：** 攻击+20%, 防御+20%, HP+15%

## 3. 战斗系统测试

### TC-201: 多单位战斗
**前置条件：** 英雄+2个生物
**步骤：**
1. 开始战斗
**预期结果：**
- 3个己方单位显示
- 按速度排序行动
- 战斗正常进行

### TC-202: 连续战斗
**前置条件：** 有生物队伍
**步骤：**
1. 完成第一关
2. 选择奖励
3. 进入第二关
4. 完成第二关
5. 选择奖励
6. 进入第三关
**预期结果：**
- 每场战斗正常开始
- 队伍数据保持一致

## 4. 数据持久化测试

### TC-301: 中途退出保存
**步骤：**
1. 选择2个生物
2. 点击返回
3. 点击继续冒险
**预期结果：**
- 生物数据恢复
- 从正确的关卡继续

### TC-302: 刷新页面
**步骤：**
1. 选择2个生物
2. 刷新页面
3. 点击继续冒险
**预期结果：**
- 生物数据恢复
- 从正确的关卡继续
